## Talend ETL 자주 사용하는 컴포넌트 TOP 20

해당 컴포넌트에 대해서는 기본적으로 기능이 실질적으로 개발되어 작동되어야한다.
아래 목록에 없는 컴포넌트 중 실전 사용빈도가 높은 컴포넌트가 있다면 업데이트 후 같이 개발 작업을 진행해야한다.

개발 순서는 구현 난이도, 코드에 미치는 영향, 컴포넌트 간 영향력을 생각하여 조절.

이미 구현된 컴포넌트의 로직과 기능은 건너띄고 진행해야한다.(중복작업 절대금지).



## 피드백 우선순위.
1. tDBRow - SELECT외 DDL/DML/Stored Procedure/Function 호출 가능
2. tREST - API 호출 연결결
3. tSystem - 커맨드라인 활용


1️⃣ 입력 (Input)

tDBInput – DB 테이블 조회

tFileInputDelimited – CSV / 구분자 파일 읽기

tFileInputExcel – Excel 파일 읽기

tRESTClient – REST API 호출(ESB)

tREST – REST API 호출(INTERNET)

tFixedFlowInput – 테스트용 데이터 생성

2️⃣ 변환 (Transformation)

tMap – 데이터 매핑 / 변환 / Join

tFilterRow – 조건 필터링

tAggregateRow – 집계 (SUM, COUNT 등)

tSortRow – 데이터 정렬

tJoin – 데이터 Join

tConvertType – 데이터 타입 변환

tReplace – 문자열 치환

3️⃣ 출력 (Output)

tDBOutput – DB 테이블 적재

tFileOutputDelimited – CSV 파일 출력

tFileOutputExcel – Excel 파일 출력

4️⃣ 제어 / 로깅 (Control & Logging)

tLogCatcher

tLogRow – 데이터 로그 출력

tWarn – 경고 로그 출력

tDie – 에러 발생 후 Job 중단

tFlowMeter – 처리 건수 측정

tChronometerStart / tChronometerStop – 실행 시간 측정

4️⃣ 기타 중요

tExtractJSONFields

## ETL Platform 구현 현황 (Talend TOP30 기준)

### 1. 실질 동작까지 구현된 컴포넌트
- **tMap (T_MAP)**: 매핑 에디터(`MappingEditorModal`), 다중 Output, 타입 경고(Tier 1) 등 포함. 고급 Enhancements/Tier 2는 일부 미구현.
- **tDBInput (T_JDBC_INPUT)**: DB 테이블 조회, 스키마 자동 로드, 증분 처리(watermark) UI/엔진 모두 구현.
- **tDBOutput (T_JDBC_OUTPUT)**: INSERT/UPDATE/UPSERT/TRUNCATE_INSERT 등 Write Mode, PK 설정까지 구현.
- **tLogRow (T_LOG_ROW)**: Row Logs 탭과 연동되어 실행 시 샘플 데이터 행 표시.
- **tFilterRow (T_FILTER_ROW)**: 조건 필터링 UI 및 실행 엔진 로직 구현.
- **tJoin (T_JOIN)**: 조인 타입/조건 설정 및 SQL Pushdown·이기종 엔진 모두에서 동작.
- **tAggregateRow (T_AGGREGATE_ROW)**: 그룹·집계 함수 설정과 실행 로직 구현.
- **tSortRow (T_SORT_ROW)**: 정렬 컬럼/순서 설정 및 실행 로직 구현.
- **tUnite (T_UNION_ROW)**: 여러 입력 합치기(UNION) 동작 구현.
- **tPrejob / tPostjob (T_PRE_JOB / T_POST_JOB)**: 잡 시작/종료 단계 제어용 Orchestration 노드 구현.
- **tDBCommit / tDBRollback (T_DB_COMMIT / T_DB_ROLLBACK)**: 트랜잭션 모드·공유 커넥션 기반 Commit/Rollback 실행까지 구현.
- **tSleep (T_SLEEP)**: Orchestration 컴포넌트로 존재하며, 실행 흐름 지연 기능 포함.

### 2. UI는 있으나 실행 엔진이 미완/부분 구현인 컴포넌트
- **tFileInputDelimited (T_FILE_INPUT)**: 캔버스/속성 패널 UI 존재. JVM FileAdapter 기반 CSV 읽기 실행 로직은 P1 로드맵 항목으로 남아 있음.
- **tFileOutputDelimited (T_FILE_OUTPUT)**: 캔버스/속성 패널 UI 존재. CSV 파일 쓰기 실행 로직은 미구현.
- **tRunJob (T_RUN_JOB)**: Orchestration 팔레트에 포함. 실제 서브 Job 호출·의존성/파라미터 전달 시맨틱은 후속 설계 필요.

### 3. Talend TOP30 기준으로 아직 미구현인 컴포넌트/기능
- **입력 계열 미구현**
  - **tFileInputExcel**: Excel 파일 입력 컴포넌트 없음.
  - **tRESTClient / tREST**: REST API 호출용 입력/출력 노드 미구현.
  - **tFixedFlowInput**: 테스트용 더미 데이터 생성 노드 미구현.
- **출력/파일·REST 유틸 계열 미구현**
  - **tFileOutputExcel**: Excel 파일 출력 노드 없음.
  - **tFileList / tFileCopy / tFileDelete**: 폴더 단위 파일 목록/복사/삭제 등 파일 관리 컴포넌트 미구현.
- **제어/로깅 계열 미구현**
  - **tLogCatcher / tWarn / tFlowMeter / tChronometerStart / tChronometerStop**: 전역 로그/경고/건수/시간 측정 전용 노드 미구현. 현재는 실행 이력·Execution Logs로 일부 대체.
- **Java/Context/JSON 계열 미구현**
  - **tJava / tJavaRow / tJavaFlex**: 임의 Java 코드를 삽입하는 컴포넌트는 존재하지 않음 (안전·일관성 위해 보류).
  - **tContextLoad**: 파일/DB 기반 컨텍스트 변수 로드는 미구현. 대신 Job IR `context`와 스케줄 `contextOverrides` 구조만 존재.
  - **tExtractJSONFields**: JSON 파싱 전용 노드 미구현.
- **DB 연결 관리 계열 미구현 (Talend 스타일 기준)**
  - **tDBConnection / tDBClose**: Talend식 수동 Connection/Close 노드는 없고, Spring + HikariCP(향후) 기반 자동 커넥션 관리 구조만 존재.
