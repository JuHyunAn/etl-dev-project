# ETL Platform 구현 로드맵

> **기준**: `구현계획기능.md` + **현재 코드베이스 분석** 결과를 반영하여, **이미 구현된 기능**과 **미구현·도입 대상**을 구분하고, 미구현 항목만 **난이도·구현 부담이 적은 순**으로 정리했습니다.


---

## 현재 구현된 기능

아래는 **이미 구현된** 기능·로직입니다. 로드맵에서는 재구현하지 않고 **확장·연결**만 진행합니다.

| 구분 | 구현 내용 |
|------|-----------|
| **인증** | JWT + Refresh Token(쿠키), OAuth2(GitHub/Google), 로컬 회원가입/로그인, Spring Security `authenticated()` |
| **자산 관리** | 프로젝트 CRUD, 잡 CRUD, Publish, **폴더 백엔드** (list/create/delete) — 폴더 **프론트 API·UI 미연결** |
| **커넥션** | CRUD, 연결 테스트, 비밀번호 암호화 저장, PostgreSQL/Oracle/MariaDB, **매 요청 DriverManager** (풀링 없음) |
| **스키마** | 테이블 목록, 컬럼 조회 (`/api/connections/{id}/schema/tables*`) |
| **Job Designer** | React Flow 캔버스, 팔레트, 속성 패널, 매핑 에디터, 스키마 트리, 컨텍스트 변수, AI Agent 패널, SQL 뷰, Execution Logs, Job Summary 탭 |
| **실행** | `POST /api/jobs/{id}/run` (context, previewMode), `POST /api/execution/preview` (previewIr) — **결과는 메모리만 반환, DB 미저장** |
| **실행 엔진** | `SqlPushdownAdapter`: T_JDBC_INPUT/OUTPUT, T_MAP/T_FILTER_ROW/T_AGGREGATE_ROW/T_SORT_ROW/T_JOIN/T_UNION_ROW/T_CONVERT_TYPE/T_REPLACE, T_DB_COMMIT/T_DB_ROLLBACK, T_LOG_ROW, Trigger 엣지(ON_OK/ON_ERROR), 트랜잭션 모드 |
| **IR/검증** | DAG 순환 검사(`hasCycle`), 위상 정렬, context 미치환 검사, `ExecutionModels`(NodeResult, ExecutionResult) |
| **DB** | `executions` 테이블 스키마 존재(V3) — **Execution 엔티티·Repository·저장 로직 없음** |
| **실행 이력 UI** | ExecutionsPage — **Store의 lastExecution만 표시**(목록 API·전체 이력 없음) |

## 현재 구현되지 않은 기능

## 1. 난이도·부담 낮음 (우선 진행)

1. **실행 이력 저장** — Execution 엔티티·Repository 추가, 실행 완료 시 `executions` INSERT, 목록/상세 API. (스키마 이미 있음.)
2. **실행 이력 목록 API + ExecutionsPage 연동** — 전체 이력 조회, 페이지네이션 또는 최근 N건. (1번 선행.)
3. **대상 DB 연결 풀링 (HikariCP)** — Connection별 DataSource 캐시, 수정/삭제 시 invalidate. (현재 `DriverManager.getConnection` 매 호출.)
4. **AI 프록시 (백엔드 경유)** — `/api/ai/chat` 등 추가, API 키 서버 보관. (선택) Resilience4j rate limiting.
5. **미리보기 결과 테이블 UI** — Preview 실행 후 `nodeResults`/샘플 행을 테이블 형태로 표시. (현재는 로그·행 수 위주.)
6. **프론트 테스트 도입 (Vitest)** — `npm i -D vitest`, `scripts.test`, CI 연동.
7. **실행 로그 DB 저장** — 실행 시 `logs` 등 `executions` 컬럼에 저장. (1번과 함께 진행 가능.)

---

## 2. 난이도·부담 중간

8. **스케줄링 (Quartz)** — Job에 `cronExpression` 추가, Quartz 연동. **선행: 실행 이력 저장.**
9. **폴더 구조 UI 연동** — 백엔드 폴더 API 있음. 프론트 `foldersApi` 추가 + ProjectDetail/사이드바에 폴더 트리 UI 연결.
10. **Expression 유효성 검사** — tMap/필터 표현식 1차 정규식·단순 규칙 → 필요 시 JSQLParser.
11. **리니지 추적 UI** — IR 파싱으로 소스→변환→타겟 그래프 생성, React Flow 전용 뷰 또는 Designer 탭.
12. **버전 관리 확장** — `job_versions` 또는 기존 version 정책, IR 스냅샷 저장. (폴더 UI와 별도 또는 연계.)
13. **T_RUN_JOB 실행 분기** — 현재 `executeNode`에서 SKIPPED. 서브잡 ID 읽어 `ExecutionService.execute` 재귀 호출.
14. **에러 처리 전략 (onError)** — 노드 config에 REJECT/FAIL/LOG 등, 실행 시 분기.
15. **RBAC (역할 기반 접근)** — User.role 활용, Spring Security 메서드 시큐리티로 Admin/Developer/Viewer 등.
16. **모니터링 (Micrometer / Actuator)** — **선행: 실행 이력 저장.** `@Observed` 등 수집 구간 한정.
17. **REJECT 라인 실행 시맨틱** — REJECT 엣지 탐지, 임시 테이블/파일 쓰기. (IR 구조는 이미 있음.)

---

## 3. 난이도·부담 높음 (단계적)

18. **파일 커넥터 (T_FILE_INPUT / T_FILE_OUTPUT)** — 속성 패널 + FileAdapter(Kotlin) 분기. SaaS 시 presigned URL 등.
19. **Pushdown 최적화 (SqlPushdownAdapter 완성)** — IR → Dialect별 SQL 컴파일, CTE/전체 파이프라인 SQL. (현재는 노드별 단순 실행 위주.)
20. **자동 스키마 변경 탐지** — 스케줄러 + 스키마 스냅샷·비교, 알림 채널.
21. **T_VALIDATE / T_PROFILE 완성** — 검증 쿼리 생성·실행, 프로파일링 노드 실행 분기. (템플릿 또는 Python API.)
22. **메타데이터 카탈로그 / 영향 분석** — PostgreSQL tsvector full-text 1차, 대규모 시 Elasticsearch 검토.
23. **데이터 보호 / 감사 추적** — PII 마스킹 규칙, AOP 감사 로그.
24. **고급 워크플로우 (조건 분기, 루프)** — IR 확장, maxIterations, 조건 평가 후 경로 선택.
25. **확장 커넥터 (NoSQL, API, 빅데이터)** — Connection 타입 확장, 메타데이터 인터페이스·구현체.
26. **혼합 엔진 (노드별 engineType)** — NodeIR에 engineType, ExecutionService에서 노드별 라우팅 (FILE/SQL 등).
27. **T_PRE_JOB / T_POST_JOB / T_SLEEP 실행** — 현재 `else -> SKIPPED`. 오케스트레이션 노드 실행 로직 추가.
28. **Spark / Airflow 연동** — SparkAdapter, Airflow HTTP 트리거.

---

## 4. 스택/도구 도입 (필요 시점에)

- **Quartz** — 스케줄링 도입 시.
- **JSQLParser** — Expression 검증 고도화 시.
- **Resilience4j** — AI 프록시 rate limiting 시.
- **Micrometer / Actuator** — 모니터링 도입 시.
- **Vault / Spring Cloud Config** — 운영 안정화·비밀 중앙 관리 시.
- **Elasticsearch / ELK** — Job·로그 규모 확대 시 단계적.
- **Ag-Grid** — 미리보기·실행 결과 대량 행/가상 스크롤 필요 시.
- **Kotest** — 팀 convention에 따라 백엔드 테스트 전환 시 (선택).

## 5. 참고

- **선행 관계**: 실행 이력 저장(1) → 실행 이력 목록(2), 스케줄링(8), 모니터링(16), 실패 알림.
- **이미 구현된 항목**은 재구현하지 말고 확장·연결만 적용 (DAG 순환 검사, 위상 정렬, executions 스키마, NodeResult, Preview API, 폴더 백엔드 등).
- 상세 적용 방향·리스크는 `구현계획기능.md` 참고.
